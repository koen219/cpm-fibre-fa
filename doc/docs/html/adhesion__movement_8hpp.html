Detailed Description

Algorithms for moving adhesions.

As the CPM evolves, cells move around and their adhesions have to be updated. This can be done in different ways, which are implemented as functions in this file.

◆ annihilation_penalty()

int annihilation_penalty ( int  num_destroyed)

Calculate annihilation penalty.

If an adhesion needs to be moved, but has nowhere to go, then it gets annihilated. This takes work (energy), the amount of which is calculated here.

Parameters
num_destroyedThe number of adhesions to be destroyed in the copy.

Find all possible adhesion displacements for a newly added pixel

If a copy attempt copies a pixel containing adhesions, then those adhesions may move along, depending on settings. This returns a list of possible displacements for those adhesions, according to an algorithm selected in the parameters.

◆ overflow_penalty()

int overflow_penalty ( int  num_present)

Calculate overflow penalty.

Stuffing too many adhesions into a single pixel takes work (energy), the amount of which is calculated here based on the parameters adhesions_per_pixel_overflow and adhesions_per_pixel_overflow_penalty.

Parameters
num_presentThe number of adhesions that will be present in the target pixel post-copy.
